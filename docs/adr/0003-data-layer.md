# ADR-0003 — لایهٔ داده: SQLite با drift + الگوی Repository

- **Status:** Accepted
- **تاریخ:** ۱۴۰۵/۰۳/۳۱
- **حاضرانِ حلقه:** `data-architect` (پیشنهاد)، `test-engineer` (چالشِ متخاصم)، جمع‌بندیِ کاندوکتور.
- **DECISION_LOG مرتبط:** مدخلِ #۰۰۳.

---

## Context

هستهٔ محصول آفلاین و بدونِ لاگین است؛ تمامِ داده روی گوشی می‌ماند. نیازهای کلیدیِ داده:
- **کوئریِ تجمیعیِ گزارش** (SUM/GROUP BY بر اساس دسته و **ماهِ شمسی**) — تمایزِ «نمودارِ خوب».
- **واکنشی‌بودن** (تغییرِ داده → به‌روزرسانیِ خودکارِ UI).
- **مهاجرتِ additive و ایمن** بدونِ فریم‌ورکِ مهاجرت (هم‌خط با فلسفهٔ پروژه).
- **پشتیبان‌گیری/بازیابیِ فایلی**، بدونِ هیچ سرویسِ گوگل.

## Decision

از **drift روی SQLite** استفاده می‌کنیم (با `sqlite3_flutter_libs` که کتابخانهٔ بومیِ sqlite را همراهِ اپ می‌آورد — بدونِ نیاز به GMS).

- **type-safe SQL + streamها:** جداول به‌صورتِ Dart تعریف می‌شوند؛ کوئریِ `watch` استریمِ زنده می‌دهد.
- **الگوی Repository:** ریپازیتوریِ `domain` (اینترفیسِ خالص) از drift بی‌خبر است؛ پیاده‌سازیِ `data` نگاشتِ ردیفِ drift ↔ مدلِ دامنه را انجام می‌دهد. **هیچ SQL خارج از `data`/`db` نیست** (اجبارِ معماری، ADR-0001 §۵).
- **مهاجرت:** `schemaVersion` + `MigrationStrategy` فقط-additive (افزودنِ جدول/ستون؛ هرگز حذف/تغییرِ مخرب)؛ روی **کپیِ دیتابیس** تست می‌شود.
- **پشتیبان‌گیری:** کپیِ فایلِ `.sqlite` یا خروجیِ JSON؛ بازیابی با جایگزینیِ امن.
- **مدلِ هدف (نرمال):** `transactions` + `categories` (FK). در **walking skeletonِ فاز ۱** برای کمینه‌کردنِ ریسک، دسته به‌صورتِ ستونِ متنی روی `transactions` است؛ **نرمال‌سازی به جدولِ `categories` نخستین کارِ فاز ۲ است** (ثبت‌شده در ROADMAP).

## Consequences

**مثبت**
- کوئریِ تجمیعیِ قوی برای گزارش‌ها (کاری که پایگاه‌های NoSQL ضعیف انجام می‌دهند).
- streamهای واکنشی مستقیماً به `StreamProvider` (ADR-0002) وصل می‌شوند.
- ایمنیِ کامپایلیِ کوئری‌ها → باگِ کمتر؛ مهاجرتِ نسخه‌دار و آزمودنی.

**منفی / هزینه**
- نیاز به `build_runner` برای تولیدِ کدِ drift (`database.g.dart`) — یک گامِ build اضافی.
- منحنیِ یادگیریِ DSLِ drift.
- مهاجرت‌ها باید با دقت و **تست روی کپی** نوشته شوند (مالکیتِ `test-engineer`).

## Alternatives Considered

| گزینه | چرا رد شد |
|-------|-----------|
| **sqflite (خام)** | بدونِ ایمنیِ نوع؛ SQLِ رشته‌ای دستی و مستعدِ خطا؛ بدونِ streamِ داخلی. |
| **Isar** | سریع، اما کوئریِ تجمیعیِ دلخواه (GROUP BY گزارش‌ها) و مهاجرتِ رابطه‌ای ضعیف‌تر؛ کتابخانهٔ بومیِ سنگین. |
| **ObjectBox** | عملکردِ خوب اما نگرانیِ مجوز/باینریِ بومی و انعطافِ کمترِ کوئریِ گزارش. |
| **Hive** | key-value؛ بدونِ کوئریِ واقعی/تجمیع؛ برای گزارش‌های مالی نامناسب. |
