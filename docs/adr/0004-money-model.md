# ADR-0004 — مدلِ پول: عددِ صحیحِ ریال + value object «Money»

- **Status:** Accepted
- **تاریخ:** ۱۴۰۵/۰۳/۳۱
- **حاضرانِ حلقه:** `data-architect` (پیشنهاد)، `test-engineer` (چالشِ متخاصم)، جمع‌بندیِ کاندوکتور.
- **DECISION_LOG مرتبط:** مدخلِ #۰۰۴.

---

## Context

پول قلبِ یک اپ مالی است. خطای ممیزِ شناور (`double`) در جمعِ تراکنش‌ها خطای انباشته می‌سازد و **غیرقابل‌قبول** است. واحدِ بانکیِ ایران **ریال** است، اما کاربر به **تومان** فکر و گفتگو می‌کند (۱ تومان = ۱۰ ریال). نمایش باید با **ارقامِ فارسی** و جداکنندهٔ هزارگان باشد.

## Decision

پول را همیشه به‌صورتِ **عددِ صحیحِ ریال (`int`)** ذخیره و محاسبه می‌کنیم؛ **هرگز `double`/`float`**.

- **value object تغییرناپذیرِ `Money`** که `int rials` را می‌پیچد. هیچ مبلغی در دامنه به‌صورتِ عدد خام جابه‌جا نمی‌شود.
- **ورودی/نمایش به تومان:** `Money.fromToman(int)` ⇒ `rials = toman * 10`؛ `money.toman ⇒ rials ~/ 10` (تقسیمِ صحیح). نمایش با ارقامِ فارسی + جداکنندهٔ هزارگان + پسوندِ «تومان».
- **عملیاتِ امن:** `+`، `-`، `* اسکالرِ صحیح`؛ بدونِ تقسیمِ از‌دست‌دهندهٔ دقت (هر سهم‌بندی با تقسیمِ صحیح + مدیریتِ باقی‌مانده، صریح).
- **علامت:** مقدارِ **منفی مجاز** است (بدهی/اصلاحیه)؛ علامت در نمایش مدیریت می‌شود.
- **سرریز:** `int` در Dart روی پلتفرمِ بومی ۶۴-بیتی است (تا ~۹.۲×۱۰¹⁸ ریال) — بسیار فراتر از هر مجموعِ واقعی؛ سرریز عملاً ناممکن.
- **ذخیره در DB:** ستونِ `amount_rial INTEGER` (ADR-0003).

## Consequences

**مثبت**
- صحتِ مطلقِ محاسبات؛ بدونِ خطای ممیزِ شناور؛ سازگار با تطبیقِ بانکی (ریال).
- یک نقطهٔ واحد برای فرمت/تجزیه → UIِ یکدست و تست‌پذیر.

**منفی / هزینه**
- هر ورودی/خروجی باید از مرزِ تومان↔ریال عبور کند؛ یک نگاشتِ ذهنیِ لازم (با `Money` کپسوله می‌شود).
- نمایشِ ریال‌های غیرمضربِ ۱۰ (نادر، چون کاربر تومان وارد می‌کند) نیاز به سیاستِ گردکردنِ صریح دارد.

## Alternatives Considered

| گزینه | چرا رد شد |
|-------|-----------|
| **`double`/`float`** | خطای انباشتهٔ ممیزِ شناور؛ برای پول ممنوع. |
| **پکیجِ `Decimal`** | دقیق اما سنگین‌تر و کندتر؛ برای واحدِ صحیحِ ریال بیش‌ازحد است. |
| **ذخیره به تومان** | دقتِ ریالی را برای تطبیقِ بانکی از دست می‌دهد؛ ریال مبنای canonical است. |
| **عددِ خام بدونِ value object** | مستعدِ قاطی‌شدنِ واحد (ریال/تومان) و فرمتِ پراکنده در کد. |
