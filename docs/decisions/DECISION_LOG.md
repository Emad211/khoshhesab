## ۰۰۱ — نامِ نهاییِ محصول — ۱۴۰۵/۰۳/۳۱
- **زمینه:** نامِ کاریِ «خوش‌حساب»؛ نیاز به نامِ متمایز از «دخل و خرج» (تاکسی‌اپس)، فارسی، مثبت، برندپذیر و ASO-friendly.
- **گزینه‌ها:** «خوش‌حساب» (اصطلاحِ مثبت: کسی که حساب‌وکتابش مرتب است) / «خرجینه» / «دخل‌یار».
- **تصمیم:** **«خوش‌حساب»** به‌عنوانِ نامِ اصلی؛ «خرجینه» و «دخل‌یار» جایگزین. (بررسیِ نهاییِ در-دسترس‌بودنِ نام در کافه‌بازار، گامِ پیش‌از‌انتشار.)
- **حاضرانِ حلقه:** `market-analyst` + `marketing-aso-advisor` (دورِ ۱).
- **پیامدها:** `package_name = khosh_hesab`؛ برندینگ/آیکون بر این نام بنا می‌شود.

## ۰۰۲ — مدیریتِ state: Riverpod — ۱۴۰۵/۰۳/۳۱
- **زمینه:** نیاز به state واکنشیِ تست‌پذیرِ جفت‌شونده با streamهای drift، کم‌سربار برای یک نفر.
- **تصمیم:** **Riverpod** (پروایدرهای دستی در اسکلت؛ generator بعداً اختیاری).
- **حاضرانِ حلقه:** `principal-architect` ✕ چالشِ `flutter-dev-ui`.
- **پیامدها / ADR:** [`adr/0002-state-management.md`](../adr/0002-state-management.md).

## ۰۰۳ — لایهٔ داده: drift/SQLite + Repository — ۱۴۰۵/۰۳/۳۱
- **زمینه:** کوئریِ تجمیعیِ گزارش، واکنشی‌بودن، مهاجرتِ additive، پشتیبان‌گیریِ فایلی، بدونِ‌GMS.
- **تصمیم:** **drift روی SQLite** + الگوی Repository؛ اسکلت با دستهٔ متنی، نرمال‌سازیِ `categories` در فاز ۲.
- **حاضرانِ حلقه:** `data-architect` ✕ چالشِ `test-engineer`.
- **پیامدها / ADR:** [`adr/0003-data-layer.md`](../adr/0003-data-layer.md).

## ۰۰۴ — مدلِ پول: ریالِ صحیح + Money — ۱۴۰۵/۰۳/۳۱
- **زمینه:** پول هرگز نباید float باشد؛ واحدِ بانکی ریال، تفکرِ کاربر تومان.
- **تصمیم:** ذخیره/محاسبهٔ **`int` ریال**، value object **`Money`**، ورودی/نمایش تومان با ارقامِ فارسی.
- **حاضرانِ حلقه:** `data-architect` ✕ چالشِ `test-engineer`.
- **پیامدها / ADR:** [`adr/0004-money-model.md`](../adr/0004-money-model.md).

## ۰۰۵ — تاریخ/زمان: میلادیِ ذخیره، شمسیِ نمایش، RTL — ۱۴۰۵/۰۳/۳۱
- **زمینه:** تجربهٔ کاملاً شمسی/RTL ولی مرتب‌سازیِ درستِ تاریخ؛ افستِ ثابتِ ایران (UTC+3:30، بدونِ DST).

- **حاضرانِ حلقه:** `principal-architect` ✕ چالشِ `qa-tester`.
- **پیامدها / ADR:** [`adr/0005-datetime-jalali.md`](../adr/0005-datetime-jalali.md).
